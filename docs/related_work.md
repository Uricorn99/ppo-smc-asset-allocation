# Related Work

## 1. 傳統資產配置與動態風險偏好

早期資產配置研究多以現代投資組合理論為基礎，透過均值–變異數框架在期望報酬與風險（變異數）之間取得權衡，並假設資產報酬近似常態分配且協方差矩陣在時間維度相對穩定。[web:18] 此類模型在平穩市場環境下能提供理論上有效的風險分散效果，但在面臨結構性變化與極端波動時，往往難以及時反映相關性重估與尾部風險放大，導致投資組合暴露於超出預期的最大回撤風險。[web:73] 此外，傳統靜態配置策略（例如固定比例股債平衡）多在事前設定風險承受度與目標比例，缺乏隨市場狀態與投資人風險偏好動態調整的彈性。[web:25]

針對上述限制，後續研究嘗試在傳統框架中導入「動態風險偏好」概念。例如，有研究以台灣 50 指數為標的，結合強化學習與投資組合理論，讓模型在觀察市場狀態變化後，自動調整 Markowitz 模型中的風險趨避參數，使資產配置權重能隨市場景氣與波動程度自適應調整。[web:18] 該類方法一方面保留均值–變異數模型的結構優點，另一方面透過學習式機制改善了傳統固定風險參數在不同市場階段下適用性不足的問題。[web:18] 儘管如此，此類研究仍多以價格與波動度等聚合指標描述市場狀態，較少明確納入市場微結構與機構流動性資訊，難以捕捉如頂部流動性掠奪、結構斷裂等細緻現象，也較難處理股、債、金同時重估的極端情境。[web:73][web:99]

## 2. 深度強化學習於資產配置

隨著深度學習與強化學習技術成熟，越來越多研究將深度強化學習（Deep Reinforcement Learning, DRL）應用於投資組合優化與資產配置問題。[web:9][web:86] 相關工作普遍採用 Actor–Critic 架構，讓代理人在離散或連續時間步中，根據觀測到的市場狀態決定各資產權重，並透過長期累積報酬最大化來學習策略。[web:9] 在演算法選擇上，近端策略最佳化（Proximal Policy Optimization, PPO）因具備樣本效率佳、更新穩定與實作相對簡潔等優點，成為處理連續動作空間（例如多資產權重分配）的主流方法之一。[web:2][web:86]

在本地研究方面，有論文將卷積神經網路（CNN）與 DDPG、PPO 結合，針對多資產投組進行配置最佳化，並設計包含平均投資報酬變化率、波動度、夏普比率、最大回撤與年均複合成長率等因子的獎勵函數，系統性比較七種獎勵設計與多種再平衡頻率對績效的影響。[web:2][web:7] 實驗結果顯示，在適當的參數與獎勵設計下，DDPG 與 PPO 模型能在平均報酬率與風險調整後指標上優於基準策略，並證實將最大回撤與交易成本懲罰納入獎勵對提升策略穩健性具有關鍵作用。[web:2] 另一篇政治大學研究則專注於比較多種深度強化學習演算法於資產配置優化問題上的表現，結果指出 PPO 在收斂穩定性與調參敏感度方面表現較佳，適合作為實務資產配置的核心演算法。[web:1][web:86]

國際研究亦提供了 DRL 在投資組合優化上的實證支持。有研究以 PPO 為基礎，在股票投組實驗中與市值加權指數進行比較，顯示 PPO 策略在允許做空情境下可達到約 22% 左右的年化報酬與約 1.1 的夏普比率，最大回撤約 −17%，在同一回測環境下明顯優於基準組合的報酬與風險表現。[web:9] 另一系列綜合性實驗則在統一的回測框架下比較傳統均值–變異數、LSTM、Transformer 與 DDPG 等方法，結果顯示 DDPG 等強化學習策略在年化報酬、夏普比率與最大回撤等指標上普遍優於傳統 Markowitz 策略，說明 DRL 在處理非線性關係與動態市場環境方面具有顯著優勢。[web:36]

此外，近期研究開始將注意力機制與圖神經網路引入資產配置領域，例如結合圖注意力（Graph Attention）的強化學習架構，以捕捉資產間動態相關性與市場結構特徵，進一步提升策略對跨標的關聯性的感知與利用能力。[web:164][web:167] 這些工作共同指出，深度強化學習不僅能在報酬與風險調整後績效上超越傳統模型，亦具備整合複雜特徵（如跨資產關聯、非線性風險因子）的展延空間。然而，現有研究多仍以價格、技術指標與基本面因子為主要特徵來源，較少直接納入市場結構與機構流動性相關的 SMC 指標作為狀態空間的一部分。[web:2][web:164][web:167]

## 3. Smart Money Concepts 與市場結構量化

聰明錢概念（Smart Money Concepts, SMC）近年在外匯、指數與加密貨幣等交易社群中受到廣泛關注，其核心思想是透過市場結構（Market Structure）、流動性區域與訂單分佈，推測銀行、對沖基金等大型機構的行為軌跡，以提高交易勝率與風險報酬比。[web:14][web:83] 與傳統依賴 RSI、MACD 等滯後指標的交易方法不同，SMC 強調從 Break of Structure（BOS）、Change of Character（CHoCh）、流動性掃蕩（Liquidity Sweep）、訂單塊（Order Block, OB）與公平價值缺口（Fair Value Gap, FVG）等現象出發，尋找價格即將反轉或加速的「結構關鍵點」，避免在價格走勢已明確化甚至接近尾聲時才進場。[web:14][web:83]

多篇實務教學與評述指出，SMC 可視為 Wyckoff 理論與訂單流分析的現代延伸：BOS 代表既有趨勢高低點結構被突破，通常用於確認趨勢延續；CHoCh 則標示趨勢由多轉空或由空轉多的性格轉變，用於辨識潛在反轉區域；OB 指出價格曾出現大額機構訂單介入的區域，往往在未來回測時提供潛在支撐或壓力；FVG 則反映價格在短時間內快速移動、成交未完全填補的失衡區域，理論上價格有較高機率在之後回補該缺口。[web:14][web:83][web:173] 透過這些結構與流動性特徵，SMC 交易者試圖在市場仍處於盤整或誘導階段時辨識機構建倉行為，並避免在流動性獵殺（如假突破）時成為被收割的對手方。[web:165][web:173]

近年亦出現將 SMC 概念部分程式化的嘗試。例如，TradingView 上的 Structure Pro+ 指標可自動標記 BOS、CHoCh、FVG 與 OB 等事件，並在價格回補 FVG 或跌破關鍵訂單塊時給出視覺化提示，顯示這些結構定義與操作規則已可轉換為明確的計算邏輯與指標組合。[web:170] 此類工具顯示 SMC 並非完全無法形式化，其市場結構元素已能被量化並用於半自動交易決策。然而，現有應用多仍侷限於單一標的或裁量交易輔助，大多數研究與實務案例尚未將 SMC 特徵系統性量化為可供強化學習模型使用的狀態向量，更少見將 FVG 距離百分比、OB 觸碰狀態與 BOS/CHoCh 方向等特徵整合進多資產配置的決策框架中。[web:14][web:170][web:165] 本研究即試圖在此缺口上前進一步，將 SMC 市場結構訊號轉換為強化學習環境中的連續觀測值，使代理人能從機構流動性視角學習動態資產配置策略。

## 4. AI 交易系統與微服務戰情室

在系統架構層面，近年金融機構與科技公司逐漸從單體式程式架構轉向微服務（Microservices）與雲原生設計，以在維持高可靠性的前提下提升開發敏捷性與系統可擴展性。微服務架構將大型應用拆分為多個小型、鬆耦合的服務，每個服務負責單一明確職責並擁有獨立資料庫，可獨立部署與水平擴充；服務之間透過 HTTP/REST 或訊息佇列進行通訊，並由 API Gateway 負責對外統一入口與授權控管。[web:169][web:171] Spring Boot 與 Spring Cloud 等框架則提供服務註冊與發現、集中設定管理、負載平衡、斷路器與監控等機制，成為 Java 生態系打造微服務架構的主流選擇。[web:166][web:171]

在交易與投資系統上，一些實作案例已將訂單管理、風險控管、報價服務與策略引擎拆分為獨立微服務，並透過事件驅動與非同步訊息機制（例如 Kafka、RabbitMQ）進行解耦，以便在高流量與低延遲的要求下維持系統穩定。[web:166] 此外，隨著雲端與前端框架的成熟，越來越多金融機構開始建置所謂的「戰情室」或「指揮中心」介面，以 React、Vue 等前端技術將即時風險曝險、資產配置狀態、策略訊號與異常警示等資訊整合於單一視覺化儀表板，協助交易員與風控人員在短時間內掌握系統整體狀況。[web:20][web:39] 這類系統不僅強調後端運算能力與資料吞吐量，也重視前端資訊呈現的效率與可解釋性。

然而，相關文獻多著重於系統工程與 DevOps 實務，較少將強化學習交易代理人、SMC 結構特徵與微服務戰情室整合為一體的完整架構。現有 AI 交易研究往往停留在單機回測或離線分析階段，缺乏將模型訓練、推論服務化與決策可視化的系統性設計。[web:4][web:12] 本研究所提出的系統架構，嘗試在此缺口上前進一步：一方面將 PPO+SMC 動態資產配置模型封裝為獨立 AI 運算服務，透過 Spring Boot API Gateway 將策略信號與歷史紀錄服務化；另一方面，以 React 建置戰情室前端，將資產配置權重、淨值曲線、風險指標與 SMC 結構事件一併呈現，以提高強化學習代理人的決策透明度與可監控性，並為未來接軌實際交易系統提供可擴充的雛形。[web:4][web:20][web:39]

# Proposed Design / 系統規格與方法設計

本章節將詳細說明本專案「動態多資產配置與微服務戰情室系統」之具體實現方法與規格設計。系統核心建立於深度強化學習 (PPO) 與聰明錢特徵 (SMC) 的融合，並透過微服務架構實踐落地化與決策視覺化。整體架構分為四大模組進行設計說明。

## 1. 多資產投資組合板塊設計 (Asset Allocation Buckets)

為應對不同市場狀態 (Regime Shifts)，系統將投資組合池劃分為三個具備不同風險屬性之板塊：
- **攻擊型板塊 (Risk-On)**：以具備高成長動能的 AI 與半導體標的為主 (如 NVDA、AMD、TSM 等)，作為承擔風險以獲取超額報酬 (Alpha) 的引擎。
- **避險型板塊 (Risk-Off)**：配置黃金 (GLD) 與長天期美債 (TLT) 等傳統防禦性資產，用於在 VIX 攀升或市場結構斷裂時承接從攻擊板塊撤出的資金，平衡波動。
- **絕對安全部位 (Safety/Cash)**：採用現金部位，當股、債、金面臨全球性流動性危機而同步下殺時，PPO 模型可選擇將權重轉移至此，以確保達成嚴格的最大回撤 (MDD) 控制目標。

## 2. 強化學習模型規格 (PPO Model Configuration)

系統採用近端策略最佳化 (Proximal Policy Optimization, PPO) 演算法作為核心代理人 (Agent)。
- **觀測空間 (State Space)**：除傳統的技術指標 (如價格、波動率) 外，著重引入**量化 SMC 特徵** 與恐慌情緒指標 (VIX)。狀態向量會標準化後送入神經網路。
- **動作空間 (Action Space)**：模型輸出一個連續向量，經過 Softmax 轉換後，決定分配給上述三個資產板塊內部標的的精確權重 (總和維持 100%)。
- **獎勵函數 (Reward Function)**：為了兼顧報酬與實務限制，獎勵函數不僅獎勵階段性總報酬，也納入了「**最大回撤懲罰**」與「**滑價及交易成本懲罰**」。確保投資組合能在不頻繁重倉的情況下追求穩定增長。

## 3. SMC 特徵量化工程 (SMC Quantification)

傳統 SMC (Smart Money Concepts) 仰賴主觀判斷，本研究將其系統化為 RL 環境的物理觀測值：
- **市場結構判定 (BOS / CHoCh)**：當價格突破前高/前低時記錄 BOS (趨勢延續) 或 CHoCh (趨勢反轉)，並轉換為 [0, 1, -1] 之離散特徵向量。
- **機構流動性熱區 (FVG & OB)**：
  - **FVG (公平價值缺口)**：計算價格與最近未填補缺口的「距離百分比」，判斷價格是否有回補吸引力。
  - **OB (訂單塊)**：記錄最近機構多/空頭訂單區塊的價位，以「碰觸狀態 (Boolean) 與距離比例」作為支撐與壓力的強度量化輸入。
透過上述特徵，模型能在機構進行流動性獵殺 (Liquidity Sweep) 或假突破時避免盲目追價，提高進出場合理性。

## 4. 微服務架構與戰情室 (Microservices & War Room)

為消彌強化學習決策的黑箱性並達到企業級穩定架構，系統採用前後端分離的雲原生微服務架構：
- **AI 運算引擎 (Python/Gymnasium)**：負責 PPO 模型的推論、回測運算與執行緒排程。
- **後端 API Gateway (Spring Boot / Java)**：作為所有微服務的進入點。內部透過 Kafka 流進行任務解耦，包含「行情資料服務」、「SMC 特徵運算服務」、「交易日誌紀錄服務」等。所有持倉與日誌安全落地於資料庫中。
- **前端戰情室 (React.js)**：建立高互動性的可視化儀表板 (War Room)。能夠即時呈現：
  1. 連續的資產配置權重轉移圖 (Stacked Area Chart)。
  2. 投資組合淨值曲線與回撤動態。
  3. K線圖上疊加的 SMC 標記 (FVG/OB/BOS)。
此介面確保交易員或風控人員可隨時監督 AI 決策邏輯並進行風險覆核，大幅提升自動化交易之信任感與系統擴展性。
